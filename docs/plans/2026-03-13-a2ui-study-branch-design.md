# A2UI 학습 브랜치 설계 문서 (초보자 중심)

## 배경
- 요청자 요구사항: 레포를 잘 모르고 Python 외 언어 경험이 적으므로, 데모를 실제 실행하면서 전체 흐름을 빠르게 이해할 수 있는 학습 브랜치가 필요함.
- 목표: "코드 안에서 바로 읽히는 설명" + "문서에서 길을 잃지 않는 내비게이션"을 동시에 제공.

## 설계 원칙
- 한글 중심: 설명은 한국어로 제공하고, 코드 식별자/표준 용어만 영어를 유지.
- 계층형 학습: 모든 파일을 같은 깊이로 설명하지 않고, 핵심 흐름 파일은 깊게, 나머지는 맵 형태로 연결.
- 실행 우선: 문서가 실제 실행 명령(`demos`)과 1:1로 연결되도록 구성.
- YAGNI: 학습과 직접 관련 없는 리팩터링/기능 변경은 하지 않음.

## 아키텍처(학습 관점)
1. 실행 계층 (`demos/`)
   - `.env` 로 모델/키 로딩
   - `run-demo*.sh` 로 에이전트 + 클라이언트 실행
   - `logs/` 에 실행 로그 기록
2. Agent 계층 (`samples/agent/adk/*`)
   - A2A 요청 수신
   - 사용자 텍스트/`userAction` 파싱
   - 툴 호출/LLM 처리 후 A2UI 파트 생성
3. Client 계층 (`samples/client/*`)
   - Flutter/Lit/React/Angular 셸에서 surface 렌더링
   - 입력 → 에이전트 → UI 업데이트 루프 확인
4. Renderer/공통 계층 (`renderers/*`, 루트 문서/설정)
   - 카탈로그/스키마/렌더러 빌드 연결

## 변경 전략
- 코드: 핵심 클래스/함수에 상세 docstring 및 주석을 추가해 "입력-처리-출력"을 바로 이해 가능하게 함.
- 문서: `docs/learning/*` 학습 경로 문서 세트를 신설해 실행 순서, 데이터 흐름, 디버깅 포인트를 단계별로 제공.
- 설정/스크립트: 초보자가 자주 헷갈리는 env/포트/실행 분기 지점에 설명 보강.

## 초보자 보조 장치
- 용어 사전 (A2A, A2UI, surface, dataModelUpdate, catalog)
- 10분/30분/반나절 학습 루트
- "이 에러가 나오면 어디를 먼저 볼지" 체크리스트
- Mermaid 기반 흐름도 (end-to-end, 실행 스크립트 오케스트레이션)

## 범위
- 범위 B: 사람이 읽는 소스/문서 + 설정 파일 전반을 학습 대상으로 포함.
- 단, lockfile/빌드 산출물 등 비학습성 파일은 수정 대신 문서에서 역할만 안내.

## 검증 계획
- 최소 1개 데모를 실제 실행하여 흐름 검증 (`restaurant-flutter` 또는 `restaurant-lit`).
- 수정 파일에 대한 문법/린트 점검 가능한 범위 수행.
- 커밋 후 PR 생성, PR 본문에 학습 경로와 테스트 방법 명시.

## 완료 정의
- 학습 브랜치에서 "처음 보는 사람"이 다음을 독립적으로 수행 가능:
  1) 데모 실행
  2) 사용자 입력이 어디를 지나 UI로 돌아오는지 설명
  3) 핵심 에러 발생 시 로그/코드 확인 지점 식별
